SQL Queries using MySQL Workbench

Overview

Throughout my Database Management course, I performed several queries as part of assignments and have included some of them below to demonstrate my knowledge and understanding of SQL databases. 

 select first_name, last_name, emp_no, count(dept_no) as totalDeptNum
 from (employees natural join dept_emp) join departments using (dept_no)
 group by first_name, last_name, emp_no
 having TotalDeptNum > 2;

This query shows the first and last name of employees who have been employed in more than 2 departments at any time during their employment.

with max_Emp(deptName,employeeCount) as (select dept_name, count(emp_no) as totalEmp
from employees natural join dept_emp natural join departments 
where from_date < curdate() and to_date > curdate()
group by dept_name)
select max(employeeCount)
from max_Emp, departments;

This query counts the number of employees in each department and show the department(s) with the most employees currently.

 with recursive ReportsTo6(EmployeeId, ReportsTo) as 
(select EmployeeId, ReportsTo 
from Employee
union
select ReportsTo6.EmployeeId, ReportsTo6.ReportsTo
from ReportsTo6, Employee
where ReportsTo6.EmployeeId = Employee.EmployeeId)
select *
from ReportsTo6
where ReportsTo6.ReportsTo = 6;

This query describes a table with one column containing employee IDs and a second column containing the ID of employees the person reports to, either directly or indirectly. From that result, the outer query selects all the IDs of the people who report to employee with ID 6, either directly or indirectly.

select Tr.Name, count(distinct P.PlaylistId)
from Track Tr, PlaylistTrack Pt, Playlist P
where Tr.TrackId = Pt.TrackId and Pt.PlaylistId = P.PlaylistId
group by Tr.Name;

This query lists the names of every track along with a count of the number of distinct Playlists that track is on.
